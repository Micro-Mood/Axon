## 1. 总览

### 设计目标

- **跨平台**：Windows / Linux / macOS 统一运行，平台差异收敛到单一抽象层
- **轻量**：1C1G 服务器可运行，无重型外部依赖
- **清晰分层**：每层职责单一，无循环依赖，依赖方向只能向下
- **安全纵深**：中间件自动拦截，handler 不需要手动调安全检查
- **可观测性**：统一的错误/警告体系，调用方能感知截断、慢操作等异常情况

### 技术栈

- Python 3.10+
- asyncio（异步 I/O）
- Pydantic 2.0+（配置验证）
- aiofiles（异步文件操作）
- structlog（结构化日志）

### 通信协议

- JSON-RPC 2.0（TCP / Stdio 两种传输模式）

```
┌──────────────────────┴───────────────────────────────────────────────┐
│                      LAYER 4: HANDLERS                               │
│  FileOps（文件读写删改）| Search（搜索）| CommandOps（命令执行）       │

                       │
┌──────────────────────┴───────────────────────────────────────────────┐
│                      LAYER 3: STREAM & BUFFER                        │
│  StreamManager + OutputBuffer                                        │
│  统一管理进程 stdout/stderr 的读取、缓冲、截断                        │
└──────────────────────┬───────────────────────────────────────────────┘
                       │
┌──────────────────────┴───────────────────────────────────────────────┐
│                      LAYER 2: PLATFORM                               │
│  platform.py — 所有平台差异的唯一出口                                 │
│  编码、信号、隐藏文件、安全默认值、文件名清理                          │
└──────────────────────┬───────────────────────────────────────────────┘
                       │
┌──────────────────────┴───────────────────────────────────────────────┐
│                      LAYER 1: CORE                                   │
│  Config | Security | Cache | Errors                                  │
│  配置加载、路径/命令校验、TTL缓存、异常树                             │
└──────────────────────────────────────────────────────────────────────┘
```

### 依赖规则

```
Layer 6 → Layer 5 → Layer 4 → Layer 3 → Layer 2 → Layer 1
                                                      ↑
                                            只能向下依赖
                                            禁止反向引用
```

### 3.1 Config — 配置管理

```python
class MCPConfig(BaseModel):
    """顶层配置，Pydantic 校验"""
    workspace: WorkspaceConfig
    security:  SecurityConfig
    performance: PerformanceConfig
    logging:   LoggingConfig

```

#### 配置加载优先级（从高到低）

```
1. 运行时 set_config() 动态修改
2. 环境变量 MCP_*
3. 配置文件 config.json
4. 代码内硬编码默认值
```

#### 配置子模块

| 子模块 | 职责 | 关键字段 |
|--------|------|---------|
| WorkspaceConfig | 工作区 | root_path, allowed_extensions, max_depth |
| SecurityConfig | 安全规则 | blocked_paths, blocked_commands, allowed_shells, max_file_size_mb |
| PerformanceConfig | 性能限制 | max_concurrent_tasks, cache_ttl, max_search_results, default_timeout_ms |
| LoggingConfig | 日志 | level, audit_enabled, log_file, log_format |
| ServerConfig | 服务器 | host, port, unix_socket, ssl_cert, ssl_key |

#### 设计要点

- 不再使用全局单例 `get_config()`
- 改为**依赖注入**：创建 handler 时传入 config
- 支持热重载：`config.reload()` 重新加载文件，通知已注册的监听者

```python
# 旧方式（AutomateX）
config = get_config()  # 全局单例

# 新方式
class FileHandler(BaseHandler):
    def __init__(self, config: MCPConfig, ...):
        self.config = config  # 注入
```

```python
class MCPError(Exception):

            "message": self.message,
            "timestamp": self.timestamp,

            "cause": str(self.cause) if self.cause else None,
        }
```

### 4.3 文件系统

```python
def is_hidden(path: Path) -> bool:
    """判断文件是否隐藏"""
    if IS_WINDOWS:
        try:
            attrs = path.stat().st_file_attributes
            return bool(attrs & stat.FILE_ATTRIBUTE_HIDDEN)
        except (AttributeError, OSError):
            pass
    return path.name.startswith(".")


def get_file_attributes(stat_info) -> dict:
    """获取平台特定文件属性"""
    result = {"is_hidden": False, "is_system": False, "is_archive": False}
    if IS_WINDOWS and hasattr(stat_info, "st_file_attributes"):
        attrs = stat_info.st_file_attributes
        result["is_hidden"] = bool(attrs & stat.FILE_ATTRIBUTE_HIDDEN)
        result["is_system"] = bool(attrs & stat.FILE_ATTRIBUTE_SYSTEM)
        result["is_archive"] = bool(attrs & stat.FILE_ATTRIBUTE_ARCHIVE)
    else:
        # Linux: 只能判断隐藏（.前缀），没有 system/archive 概念
        pass
    return result


def sanitize_filename(filename: str) -> str:
    """清理文件名，移除平台不允许的字符"""
    if IS_WINDOWS:
        forbidden = r'<>:"/\|?*' + "".join(chr(i) for i in range(32))
        reserved = {"CON", "PRN", "AUX", "NUL",
                     "COM1","COM2","COM3","COM4","COM5","COM6","COM7","COM8","COM9",
                     "LPT1","LPT2","LPT3","LPT4","LPT5","LPT6","LPT7","LPT8","LPT9"}
        result = filename
        for c in forbidden:
            result = result.replace(c, "_")
        result = result.rstrip(". ")
        if Path(result).stem.upper() in reserved:
            result = f"_{result}"
        return result or "unnamed"
    else:
        # Linux: 只禁止 / 和 \0
        return filename.replace("/", "_").replace("\0", "_") or "unnamed"
```

## 5. Layer 3: Stream & Buffer（流管理层）

### 5.1 OutputBuffer — 输出缓冲区

每个进程的每个流（stdout/stderr）对应一个 OutputBuffer 实例。

```python
class OutputBuffer:
    """
    统一输出缓冲区

    特性:
    - 写入时自动解码（通过 platform.decode_output）
    - 硬性大小限制，超限自动截断
    - 支持多消费者（每个 reader 维护独立游标）
    - 提供消费式读取和非破坏性 peek
    """

    def __init__(self, name: str, max_size: int = 10 * 1024 * 1024):
        self._name: str = name          # "stdout" 或 "stderr"
        self._max_size: int = max_size  # 最大字节数 (默认 10MB)

        # 数据存储
        self._raw: bytearray = bytearray()  # 原始字节
        self._decoded: str = ""              # 解码后文本

        # 状态
        self._total_received: int = 0   # 累计接收字节（含被丢弃的）
        self._truncated: bool = False   # 是否已触发截断
        self._eof: bool = False         # 流是否结束

        # 多消费者游标
        self._readers: dict[str, int] = {}  # reader_id → 已读字符位置

    # ── 写入端 ──

    def write(self, data: bytes) -> int:
        """
        写入原始字节数据

        流程:
        1. 累计计数 += len(data)
        2. 如果已截断 → 丢弃，返回 0
        3. 检查剩余空间，不够则截断
        4. 追加到 _raw
        5. 调用 platform.decode_output() 解码
        6. 追加到 _decoded

        Returns:
            实际写入的字节数
        """
        ...

    def mark_eof(self) -> None:
        """标记流结束"""
        self._eof = True

    # ── 读取端 ──

    def read(self, reader_id: str = "default", max_chars: int = 8192) -> str:
        """
        消费式读取

        每个 reader_id 维护独立游标。
        读过的内容不会从 buffer 中删除（与 AutomateX 的区别）。
        多个消费者可以各自独立消费。

        Returns:
            最多 max_chars 个字符的文本
        """
        pos = self._readers.get(reader_id, 0)
        chunk = self._decoded[pos : pos + max_chars]
        self._readers[reader_id] = pos + len(chunk)
        return chunk

    def peek(self, max_chars: int = 8192) -> str:
        """非破坏性读取（不移动任何游标）"""
        return self._decoded[:max_chars]

    def drain(self, reader_id: str = "default") -> str:
        """读取全部剩余内容"""
        pos = self._readers.get(reader_id, 0)
        chunk = self._decoded[pos:]
        self._readers[reader_id] = len(self._decoded)
        return chunk

    def get_all(self) -> str:
        """获取全部已解码内容（忽略游标）"""
        return self._decoded

    def get_raw_base64(self) -> str:
        """获取原始字节的 base64 编码"""
        import base64
        return base64.b64encode(bytes(self._raw)).decode("ascii")

    # ── 状态查询 ──

    @property
    def eof(self) -> bool:
        return self._eof

    @property
    def truncated(self) -> bool:
        return self._truncated

    @property
    def size(self) -> int:
        """当前缓冲区字节数"""
        return len(self._raw)

    @property
    def total_received(self) -> int:
        """累计接收字节数（含截断丢弃的）"""
        return self._total_received

    def unread_count(self, reader_id: str = "default") -> int:
        """指定 reader 还有多少字符未读"""
        pos = self._readers.get(reader_id, 0)
        return len(self._decoded) - pos
```

```python
                if not chunk:

        """
        ...

    async def finalize(self, task_id: str) -> dict:
        """
        进程结束后收尾

        1. 等待 reader 协程结束（确保数据读完）
        2. 标记 eof
        3. 返回最终结果:
           {
               "stdout": buffer_stdout.get_all(),
               "stderr": buffer_stderr.get_all(),
               "stdout_raw_b64": buffer_stdout.get_raw_base64(),
               "stderr_raw_b64": buffer_stderr.get_raw_base64(),
           }
        """
        ...

    def cleanup(self, task_id: str) -> None:
        """释放缓冲区内存"""
        self._streams.pop(task_id, None)
        self._reader_tasks.pop(task_id, None)

    # ── 读取接口 ──

    def get_buffer(self, task_id: str, stream: str) -> OutputBuffer:
        """获取指定流的 buffer（"stdout" 或 "stderr"）"""
        ...

    def read(self, task_id: str, stream: str,
             reader_id: str = "default", max_chars: int = 8192) -> str:
        """便捷读取"""
        return self.get_buffer(task_id, stream).read(reader_id, max_chars)

    def drain(self, task_id: str, stream: str,
              reader_id: str = "default") -> str:
        """便捷 drain"""
        return self.get_buffer(task_id, stream).drain(reader_id)
```

### 5.3 数据流图示

```
进程启动
    │
    ▼
StreamManager.start(task_id, process)
    │
    ├─ 创建 OutputBuffer("stdout", max=10MB)
    ├─ 创建 OutputBuffer("stderr", max=10MB)
    │
    ├─ 启动 _read_loop(process.stdout, buf_stdout)
    │       │
    │       ▼ (循环)
    │       chunk = await process.stdout.read(4096)  ← 4KB 块读取
    │       if not chunk: break                       ← EOF 退出
    │       buf_stdout.write(chunk)                   ← 解码 + 存储
    │           │
    │           ├─ _raw.extend(decoded_bytes)
    │           ├─ _decoded += platform.decode_output(chunk)
    │           └─ 超限? → _truncated = True, 后续丢弃
    │       (循环结束)
    │       buf_stdout.mark_eof()
    │
    └─ 启动 _read_loop(process.stderr, buf_stderr)
            └─ (同上)

读取请求
    │
    ▼
handler.read_stdout(ctx, task_id, max_bytes=8192)
    │
    ▼
stream_mgr.read(task_id, "stdout", reader_id=ctx.request_id, max_chars=8192)
    │
    ▼
OutputBuffer.read(reader_id, max_chars)
    │
    ├─ pos = _readers[reader_id]         ← 该 reader 上次读到哪
    ├─ chunk = _decoded[pos:pos+8192]    ← 取出新内容
    ├─ _readers[reader_id] += len(chunk) ← 移动游标
    └─ return chunk

进程结束
    │
    ▼
await stream_mgr.finalize(task_id)
    │
    ├─ await 所有 reader 协程
    ├─ mark_eof()
    └─ return {stdout, stderr, raw_b64...}

清理
    │
    ▼
stream_mgr.cleanup(task_id)
    └─ 释放 buffer 内存
```

### 6.5 Task 数据模型

```python
class TaskState(Enum):
    CREATED   = "created"
    RUNNING   = "running"
    COMPLETED = "completed"
    FAILED    = "failed"
    STOPPED   = "stopped"   # 优雅停止
    KILLED    = "killed"    # 强制终止
    TIMED_OUT = "timed_out" # 超时终止

@dataclass
class Task:
    task_id: str
    command: str
    cwd: str | None
    env: dict | None
    state: TaskState
    process: asyncio.subprocess.Process | None

    pid: int | None = None
    exit_code: int | None = None
    signal: str | None = None

    created_at: datetime
    started_at: datetime | None = None
    completed_at: datetime | None = None

    @property
    def duration_ms(self) -> float | None:
        if self.started_at and self.completed_at:
            return (self.completed_at - self.started_at).total_seconds() * 1000
        return None

    @property
    def is_active(self) -> bool:
        return self.state in (TaskState.CREATED, TaskState.RUNNING)
```

```
CREATED ──spawn()──→ RUNNING ──(正常退出)──→ COMPLETED
                       │
                       ├──(exit_code≠0)──→ FAILED
                       ├──stop()─────────→ STOPPED
                       ├──kill()─────────→ KILLED
                       └──(超时)─────────→ TIMED_OUT
```

### 7.2 RequestContext

```python
@dataclass
class RequestContext:
    """每个请求一个实例，贯穿中间件链和 handler"""

    request_id: str | int       # JSON-RPC request id
    method: str                 # 方法名，如 "read_file"
    params: dict                # 原始参数
    start_time: datetime        # 请求开始时间
    warnings: list[Warning]     # handler/middleware 随时可 append

    # 安全中间件填入
    validated_paths: dict[str, Path] = field(default_factory=dict)
    # key = 参数名（"path", "source", "destination"）
    # value = 验证后的绝对路径

    def warn(self, code: str, message: str, **details) -> None:
        """添加警告"""
        self.warnings.append(Warning(code=code, message=message, details=details))

    @property
    def duration_ms(self) -> float:
        return (datetime.now(timezone.utc) - self.start_time).total_seconds() * 1000
```

### 7.3 SecurityMiddleware

```python
class SecurityMiddleware:
    """
    自动安全校验

    根据方法名确定需要校验哪些参数:
    - 含 path/source/destination 参数 → validate_path
    - 含 command 参数 → validate_command
    - 含 root_dir 参数 → validate_path

    handler 不需要手动调安全检查。
    """

    # 路径参数名映射
    PATH_PARAMS = {"path", "source", "destination", "root_dir", "root"}

    # 命令参数名映射
    COMMAND_PARAMS = {"command"}

    async def __call__(self, ctx: RequestContext, next_handler):
        # 自动校验所有路径参数
        for param_name in self.PATH_PARAMS:
            if param_name in ctx.params:
                validated = self.security.validate_path(
                    ctx.params[param_name],
                    workspace=self.workspace_path,
                )
                ctx.validated_paths[param_name] = validated
                ctx.params[param_name] = str(validated)  # 替换为绝对路径

        # 自动校验命令参数
        for param_name in self.COMMAND_PARAMS:
            if param_name in ctx.params:
                self.security.validate_command(ctx.params[param_name])

        return await next_handler(ctx)
```

### 7.4 AuditMiddleware

```python
class AuditMiddleware:
    """
    审计日志 + 慢操作警告
    """

    SLOW_THRESHOLD_MS = 5000  # 超过 5s 算慢

    async def __call__(self, ctx: RequestContext, next_handler):
        result = await next_handler(ctx)

        duration = ctx.duration_ms
        if duration > self.SLOW_THRESHOLD_MS:
            ctx.warn(
                "SLOW_OPERATION",
                f"操作耗时 {duration:.0f}ms 超过阈值 {self.SLOW_THRESHOLD_MS}ms",
                duration_ms=duration,
                threshold_ms=self.SLOW_THRESHOLD_MS,
            )

        logger.info(
            "operation_complete",
            method=ctx.method,
            duration_ms=round(duration, 2),
            warnings_count=len(ctx.warnings),
            status="success",
        )

        return result
```

## 9. 错误 & 警告体系

### 9.1 错误分级

```
Level 0: FATAL — 服务不可用
    • ConfigLoadError   配置文件损坏
    • PortInUseError    端口被占
    → 行为: 拒绝启动，日志 CRITICAL，退出进程

Level 1: ERROR — 本次请求失败
    • 所有 MCPError 子类
    → 行为: 返回 JSON-RPC error，日志 ERROR

Level 2: WARNING — 请求成功但有隐患
    • OUTPUT_TRUNCATED   输出超 10MB 被截断
    • SLOW_OPERATION     操作耗时 > 5s
    • LARGE_FILE         文件 > 50MB
    • HIGH_MEMORY        内存使用率 > 80%
    • PARTIAL_RESULT     搜索命中上限，结果不完整
    • DEPRECATED_METHOD  方法即将弃用
    → 行为: 请求成功，result 内带 warnings 数组

Level 3: INFO — 正常审计
    → 行为: 仅写日志
```

## 10. 请求生命周期

完整的一次 `read_file` 请求：

```
1. [Protocol]    TCP 收到 JSON 字符串
2. [Protocol]    解析 JSON-RPC，提取 method="read_file", params={path: "data/notes.md"}
3. [Protocol]    _coerce_params() 类型转换
4. [Protocol]    创建 RequestContext(method="read_file", params=...)
5. [Middleware]  SecurityMiddleware:
                   validate_path("data/notes.md", workspace="/home/user/memo")
                   → 绝对路径 /home/user/memo/data/notes.md
                   → 检查不在 blocked_paths 中 ✓
                   → 检查在 workspace 内 ✓
                   → ctx.validated_paths["path"] = Path(...)
                   → ctx.params["path"] = "/home/user/memo/data/notes.md"
6. [Middleware]  ValidationMiddleware: 参数完整性检查 ✓
7. [Middleware]  RateLimitMiddleware: 未超限 ✓
8. [Handler]    FileHandler.read_file(ctx, path=..., encoding="utf-8")
                   → cache.get("metadata", path) → miss
                   → aiofiles.open(path) → read content
                   → cache.set("metadata", path, {...})
                   → return {"path": ..., "content": ..., "size": ...}
9. [Middleware]  AuditMiddleware:
                   → duration = 15ms < 5000ms → 不告警
                   → log: method=read_file, duration=15ms, status=success
10. [Protocol]   格式化: {"jsonrpc": "2.0", "result": {"status": "success", ...}}
11. [Protocol]   TCP 发送响应
```

完整的一次 `run_command` 请求：

```
1. [Protocol]    收到 {method: "run_command", params: {command: "ls -la", timeout: 5000}}
2. [Protocol]    _coerce_params() → timeout: "5000" → 5000
3. [Protocol]    创建 RequestContext
4. [Middleware]  SecurityMiddleware:
                   validate_command("ls -la")
                   → 不在 blocked_commands 中 ✓
5. [Handler]    CommandHandler.run(ctx, command="ls -la", timeout=5000)
                   │
                   ├─ spawn():
                   │   process = create_subprocess_shell("ls -la", ...)
                   │   stream_mgr.start(task_id, process)  ← 开始收集输出
                   │
                   ├─ wait(task_id, timeout=5000):
                   │   await process.wait() or timeout
                   │
                   ├─ stream_mgr.finalize(task_id):
                   │   ← 等待 reader 协程结束
                   │   ← 返回 {stdout, stderr}
                   │
                   ├─ 检查截断:
                   │   if stream_mgr.get_buffer(task_id, "stdout").truncated:
                   │       ctx.warn("OUTPUT_TRUNCATED", ...)
                   │
                   ├─ stream_mgr.cleanup(task_id)
                   │
                   └─ return {exit_code, stdout, stderr, duration_ms}
6. [Middleware]  AuditMiddleware: log
7. [Protocol]   格式化响应（含 warnings 如果有）
```

```
  │                            │                           │   创建 stdout buf    │
  │                            │                           │   创建 stderr buf    │
  │                            │                           │   启动 reader 协程   │
  │                            │                           │       │               │
  │                            │── wait() ────────────────→│       │ read(4096)    │
  │                            │   (等待进程退出)            │       │──────────────→│
  │                            │                           │       │ write(chunk)  │
  │                            │                           │       │──────────────→│
  │                            │                           │       │ (循环...)     │
  │                            │                           │       │               │
  │                            │   (进程退出)               │       │ EOF           │
  │                            │                           │       │── mark_eof()─→│
  │                            │                           │                      │
  │                            │── finalize() ────────────→│                      │
  │                            │   ← {stdout, stderr}      │── get_all() ────────→│
  │                            │                           │   ← 全部文本         │
  │                            │                           │                      │
  │                            │── cleanup() ─────────────→│                      │
  │                            │                           │── 释放内存           │
  │                            │                           │                      │
  │←── {exit_code, stdout...} ─│                           │                      │
```

### 11.2 异步执行 (spawn + stream)

```
调用方                     CommandHandler              StreamManager          OutputBuffer
  │                            │                           │                      │
  │── spawn() ────────────────→│                           │                      │
  │←── task_id ────────────────│── start() ───────────────→│                      │
  │                            │                           │── reader 持续读取 ──→│
  │                            │                           │                      │
  │                            │                           │ (进程还在跑...)       │
  │                            │                           │                      │
  │── read_stdout(task_id) ───→│                           │                      │
  │                            │── read(task_id, stdout) ─→│                      │
  │                            │                           │── read(reader_id) ──→│
  │                            │                           │←── "部分输出..." ────│
  │←── {output, eof: false} ──│                           │                      │
  │                            │                           │                      │
  │── read_stdout(task_id) ───→│  (再读一次)                │                      │
  │                            │── read() ────────────────→│── read(reader_id) ──→│
  │                            │                           │←── "更多输出..." ────│
  │←── {output, eof: false} ──│                           │                      │
  │                            │                           │                      │
  │── stop(task_id) ──────────→│                           │                      │
  │                            │── platform.send_interrupt │                      │
  │                            │── await process.wait()    │                      │
  │                            │── finalize() ────────────→│── mark_eof() ───────→│
  │                            │                           │                      │
  │←── {state: stopped} ──────│                           │                      │
  │                            │                           │                      │
  │── read_stdout(task_id) ───→│  (最后一次读取)            │                      │
  │                            │── drain() ───────────────→│── drain() ──────────→│
  │←── {output, eof: true} ───│                           │←── "剩余全部" ───────│
```

```
调用方                     CommandHandler              进程
```

### 11.4 超时处理

```
spawn(command, timeout=5000)
    │
    ├─ asyncio.create_task(_timeout_monitor(task, 5000))
    │
    │   _timeout_monitor:
    │       await asyncio.sleep(5.0)
    │       if task.state == RUNNING:
    │           platform.force_kill(process)  ← 超时强杀
    │           task.state = TIMED_OUT        ← 区别于 KILLED
    │           stream_mgr.finalize(task_id)
    │
    └─ (正常退出则 cancel 掉 monitor)
```

## 13. 与 AutomateX 的对比

| 维度 | AutomateX 现有 | 重构后 |
|------|---------------|--------|
| **分层** | 3 层（Protocol → Handler → Core） | **6 层**，职责更清晰 |
| **平台代码** | 散落在 4 个文件里 | **Layer 2 Platform 统一收敛** |
| **安全检查** | handler 手动调 `security.validate_path()` | **Middleware 自动拦截**，handler 不感知 |
| **依赖方式** | 全局单例 `get_config()` / `get_handler()` | **依赖注入**，构造时传入 |
| **错误** | 18 种异常，扁平排列 | **分树**：RequestError(4xx) / ServerError(5xx) |
| **警告** | ❌ 只写日志，调用方不知道 | ✅ **warnings 数组**嵌入响应 |
| **流管理** | reader 直接写 task 字段 | **StreamManager + OutputBuffer** 独立层 |
| **消费模型** | 单消费者，读了就删 | **多消费者游标**，读不删 |
| **combined_output** | 无限制，可能 OOM | OutputBuffer 统一 10MB 限制 |
| **attach/detach** | 只改标志位 | 实际注册/注销 reader |
| **信号处理** | 硬编码 CTRL_C_EVENT | **platform.send_interrupt()** |
| **文件名清理** | SecurityManager 兼职 | **platform.sanitize_filename()** |
| **超时状态** | 超时后 state=KILLED | **新增 TIMED_OUT** 状态，可区分 |
| **文件结构** | core/ + modules/ 两层 | **6 个目录**对应 6 层 |
| **中间件** | ❌ 没有 | ✅ **链式 Security → Validation → RateLimit → Audit** |
| **RequestContext** | ❌ 没有 | ✅ **贯穿全程**，携带 warnings、validated_paths |
| **配置热重载** | 重置所有 handler 单例 | **config.reload()** + 监听器通知 |
